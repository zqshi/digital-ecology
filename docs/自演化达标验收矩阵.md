# 自演化达标验收矩阵（机制 × 指标 × 阈值 × 数据源 × 责任人）

## 1. 使用方式

本矩阵用于判定生态系统是否具备并维持“自演化能力”。
结论分级：
- 绿色（达标）：关键阈值全部满足，可扩大规模。
- 黄色（受限）：存在1-2项关键阈值偏离，只允许灰度运行。
- 红色（失稳）：任一红线指标触发，立即冻结扩张并进入修复期。

评审节奏：
- 周评审：运行与风险指标。
- 双周评审：经济与协作指标。
- 月评审：治理与制度演化指标。

---

## 2. 总体验收门槛（Go/No-Go）

| 维度 | 核心门槛（连续4周） | 结论 |
|---|---|---|
| 升级可靠性 | 升级成功率 >= 85%，回滚率 <= 15% | 不满足则禁止放量 |
| 安全稳定性 | L4 事故 = 0，L3 事故 <= 1次/周 | 不满足则冻结扩张 |
| 治理有效性 | 制度提案生效率 >= 40%，观察期通过率 >= 70% | 不满足则治理流程整改 |
| 经济闭环 | 契约结算闭环率 >= 70%，违约追偿率 >= 90% | 不满足则暂停高价值任务 |
| 生态活力 | 多样性指数不下降（环比 >= 0）且协作收益环比 >= 5% | 不满足则调整激励结构 |
| 自主进化性 | 自主目标率 >= 20%，被动化指数 <= 40% | 不满足则触发反被动修复 |
| 人类交付能力 | 按期交付率 >= 85%，一次验收通过率 >= 70% | 不满足则限流外部任务 |
| 数字考古质量 | 样本覆盖率 >= 80%，制度模板生效率 >= 30% | 不满足则暂停制度放量 |

## 4. 达标依赖检查清单（前置条件）

在验收矩阵执行前，必须先满足以下“依赖完备”：

| 依赖项 | 验收条件 | 负责人 |
|---|---|---|
| 身份账本 | EID、公钥、责任实体、版本指纹四元组可查询 | 身份治理组 |
| 审计存储 | WORM策略启用，关键事件不可篡改 | 安全与合规组 |
| 策略执行链 | PDP/PEP全链路生效且可观测 | 安全策略组 |
| 事件总线 | 关键服务事件统一Schema并上报成功率>=99% | 平台基础组 |
| 仲裁流程 | 申诉、复核、执行三阶段流程可运行 | 治理委员会 |
| 应急编排 | L3/L4处置剧本已演练并通过 | IR团队 |
| 演化服务 | 候选生成、实验编排、回滚链路可运行 | 演化工程组 |
| 自主预算 | 三池预算隔离（生存/履约/自主）已启用 | 财政治理组 |
| 考古管线 | ETL、去偏、证据评分、模板注册链路可运行 | 数据治理组 |

任一依赖项不达标，矩阵结论自动降级为“黄色受限”。

## 5. 90天验收执行计划

### 第1-2周：基线建立
- 固化指标口径与事件Schema。
- 建立周报/双周报/月报自动化面板。
- 完成一次全链路应急演练。

### 第3-6周：小规模压力运行
- 在Open Zone引入外生Agent，执行低风险任务。
- 跑通升级闸门、隔离流程、契约结算闭环。
- 生成首轮红黄绿评估。

### 第7-10周：生产灰度
- 扩展到Work Zone，开启中等价值任务。
- 执行首次制度修订（含双通道表决+观察期）。
- 复盘误伤率、回滚率、修复成功率。

### 第11-12周：Go/No-Go评审
- 对照第2章总体验收门槛给出结论。
- 若绿色：进入放量阶段。
- 若黄色/红色：输出整改清单并冻结对应扩展动作。

## 6. 结论判定模板（评审会直接使用）

- 结论等级：绿色 / 黄色 / 红色
- 本期触发红线：是 / 否
- 不达标项：
  - 项目A：现状 -> 目标 -> 修复动作 -> 责任人 -> 截止日期
  - 项目B：现状 -> 目标 -> 修复动作 -> 责任人 -> 截止日期
- 是否允许下一阶段扩展：允许 / 有条件允许 / 不允许

---

## 7. 演化停机线（Hard Stop）

以下任一条件触发，立即进入“冻结扩张 + 强制整改”：
- 连续2周自主目标率 < 10%。
- 连续2周被动化指数 > 60%。
- 演化试验连续3个周期通过率 < 10%。
- 同类失败事件在30天内重复出现 >= 3次且无制度修订生效。
- 连续2周按期交付率 < 70% 或一次验收通过率 < 50%。
- 连续2个月样本覆盖率 < 60% 或证据包完备率 < 90%。

整改完成后需通过一次完整Go/No-Go复评方可恢复扩张。
